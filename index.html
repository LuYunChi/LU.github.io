MessageFill

Posted on 2019-01-09

For organizers, it‚Äôs common to send information message to considerable recipients, but the message only varies in a few specific keys: name, activity time, location, etc. So, this app is a productivity tool to help you focus on phone numbers and other different parts. It also simplifies the procedure of mass texting. To make this app as professional as possible, I also deal with error inputs and data matching problems.

Preview

How to Use

  1. Input the template message, with words to be replaced marked by unique keys.
  2. Tap a color button to activate the corresponding key, and fill the blanks with keys. Afterwards, press the red eye-looking match button, so that, if keys are valid, match button will turn green, also keys will be highlighted in the template message.
  3. Now you can fill out information, and press send button. Message sending window will pop out automatically, and status can be viewed after sending the message.
  4. After a message is sent, there‚Äôll be an extra recipient info box. Also, you can press ‚Äú+‚Äù to append a recipient box, or press the bin to clear all boxes.

How to Install


Since Apple‚Äôs developer account is not free for students, this app is not available on App Store. If you wanna use it, follow these steps:

  1. Install Xcode in Mac OS, and open ‚ÄúMessageFill.xcodeproj‚Äù in the root content.
  2. Click Simulator in the top bar.
  3. Choose your device. Make sure that your phone is connected to the computer. Then click run.
  4. Trust the developer in your phone settings and install the app.

About Handling Misoperation

To make the app function stably in real application, we need to take all invalid inputs into account. The match button, red eye-looking one, only turns green when keys inputs conform to following rules (the sequence is based on hierachy), else recipients‚Äô info cannot be filled and the button will remain red, popping out an alert window to tell user which rule is violated.
Rule 1: At least one key blank is activated and filled.
Rule 2: There‚Äôs no activated blank remain unfilled.
Rule 3: There‚Äôre no same keys.
Additionally, every time the user changes letters in template message or keys, we require him to press match button before making changes in recipients‚Äô info.

In this app, user may change keys when filling the recipients‚Äô info. And every time we reload the data of UITableView, we want to remain the old, filled recipients, as well as vary with changed keys. So, as a table cell is created, an accompanied instance of class Member is created also; moreover, the textfields, labels are matched to the instance ‚Äî‚Äî it‚Äôs difficult to access data through views of UITableViewCell, but feasible to access through class members. Retrospect & Prospect

In fact, at the beginning I designed the app to be able to send different messages within one click; however, it seems that Apple doesn‚Äôt open such interface to developers. Though user should click send for every recipient now, it still largely simplifies the procedure of copying, pasting, selecting, and changing specific words in a message. So the effeciency is increased, and possibility for misinputs is reduced.
To make it more productive, I may consider adding functions of:

  2. recipients‚Äô info can be derived through external apps, such Excel or WPS; but it necessitates more formatting constraints

About NSTimer & GCD Timer‚≠êÔ∏è

In most Github examples, NSTimer is used to keep the time; however, you may find the app stops keeping the time when you scroll the table.

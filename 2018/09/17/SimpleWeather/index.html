SimpleWeather

Posted on 2018-09-17

Searching for Swift applications on the Internet, you may find building a weather forecast application a traditional project. It involves requiring JSON data from the Internet, and more importantly analyzing data, in other words, match the JSON data with our customed structures. Within this project, I also applied simple animation and row-action of UITableView, as well as UIAlertController. Devoid of professional weather data api (since those are not available for free), the app is just a demo; but techniques are the same.

Remarks

The workload of this project is substantial for newcomers, so we can follow these steps:

  1. Acquire and analyze weather data of a specific city.
  2. Realize the visual components based on the derived data.
  3. Add “+” button and realize the function of acquiring data of different cities. This part involves using alert window and handling error city inputs.
  4. Decorate the app with animations and test the application.

About AlertController & RunLoop⭐️

If we are to present an alert window and want it to disappear automatically after a time interval, NSTimer is required. But we demand the thread of the timer does not conflict with and suspended by other threads. Apart from the method of using GCD Timer,which has been introduced in UltraTimer, setting RunLoop also reaches the same effect, in a more precise manner.

Reflection

This demo app is not delicate enough, and can be improved in following derections later on:
-More professional api, with more complicated view arrangements can be applied.
-Current weather icon set does not cover all weather conditions, and should be modified to cater to unusual weather conditions.
-We now ask the user to input a city or region, which results in much uncertainty. So another UIView to present city options where users can search and select their location will be helpful.
-Actually the source code is written in a messy manner: non-systematic nomenclature of variables, repeated assignment operation that can be done by loop, etc. It “could”😂 be written more tidy.
